# Сценарии приложения Нотсапп

## **Пользовательский сценарий «Просмотр заметки»**

*Действующие лица*:

1. Пользователь.

2. Приложение.

3. База данных.

!!! warning "Внимание"
    Предварительные условия: пользователь находится на главном экране.

???+ note "Выходные условия"
    На экране отображается текст заметки.

**Основной сценарий**

* Пользователь нажимает на заметку 1.

* Приложение открывает окно с текстом заметки в режиме просмотра, текст заметки подтягивается из Базы данных (в базе данных хранятся пользовательские заметки).

* Пользователь просматривает заметку.

* Пользователю доступны кнопки **Редактировать**, **Удалить**, **мои заметки** под окном с текстом заметки.

* Для возврата к списку всех заметок пользователь нажимает на кнопку **мои заметки**.

* Приложение открывает главную страницу со списком всех заметок.

## **Пользовательский сценарий «Создание новой заметки»**

*Действующие лица*:

1. Пользователь.

2. Приложение.

3. Бэк.

4. База данных.

<span style="color:green">Предварительные условия</span>

Пользователь должен находиться на главном экране.

<span style="color:green">Выходные условия</span>

В системе появилась новая заметка пользователя.

**Основной сценарий**

* Пользователь нажимает кнопку **Создать новую**.

* Приложение открывает окно с формой создания новой заметки.

* Пользователь заполняет форму заметки.

* Пользователь нажимает кнопку **Сохранить**.

* Приложение отправляет запрос Бэку на сохранение новой заметки.

* Бэк сохраняет заметку в Базе данных.

* Бэк возвращает Приложению ответ об успешном сохранении заметки.

* Приложение открывает пользователю уведомление «Заметка успешно сохранена».

**Альтернативный сценарий**

* Пользователь нажимает кнопку **Создать новую**.

* Приложение открывает окно с формой создания новой заметки.

* Пользователь заполняет форму заметки.

* Пользователь нажимает кнопку **Отменить**.

* Приложение открывает пользователю главный экран.
